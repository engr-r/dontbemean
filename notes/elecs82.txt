# Microprocessor & Microcontroller Systems and Design: Memory Subsystem

## 1. Knowledge (Recall & Facts)

### Definitions

* **Memory Subsystem**: Storage architecture; holds instructions/data for MPU execution.
* **Volatile Memory**: Loses data upon power removal; temporary (RAM).
* **Non-Volatile Memory (NVM)**: Retains data without power; permanent (ROM, Flash).
* **RAM (Random Access Memory)**: Read/write memory; equal access time for any location.
* **SRAM (Static RAM)**: Flip-flop transistor pair design; retains data without refresh; fast; low density.
* **DRAM (Dynamic RAM)**: Capacitor-based design; requires continuous electrical refresh; high density; slower.
* **ROM (Read-Only Memory)**: Mask-programmed during manufacturing; immutable.
* **EEPROM**: Electrically Erasable Programmable ROM; byte-level erasable; limited write cycles.
* **Flash Memory**: Sector/block-level erasable EEPROM variant; high density (NOR/NAND).
* **Cache**: Ultra-fast SRAM located near/inside MPU; buffers frequently used data.
* **Memory Mapping**: Assigning logical MPU addresses to physical memory chips/IO ports.
* **Memory Decoder**: Logic circuit (e.g., 74LS138); translates higher-order address lines into Chip Select ($\overline{CS}$) signals.

### Critical Numbers

* **DRAM Refresh Rate**: Typically $2\text{ ms}$ to $64\text{ ms}$; standard interval requires memory controller intervention.
* **Typical SRAM Latency (L1 Cache)**: $1\text{ ns}$ to $5\text{ ns}$.
* **Typical DRAM Latency (Main Memory)**: $50\text{ ns}$ to $100\text{ ns}$.
* **EEPROM/Flash Write Endurance**: $10,000$ to $100,000$ cycles per sector (standard); up to $1,000,000$ (industrial).
* **Memory Hierarchy Speeds**: Registers ($<1\text{ ns}$) > L1 Cache ($\sim 1\text{ ns}$) > L2 Cache ($\sim 3\text{ ns}$) > DRAM ($\sim 60\text{ ns}$) > Flash/SSD ($\sim 100\text{ \mu s}$).

### Standards & Protocols

* **JEDEC**: Defines standard timing, voltages, and architectures for DRAM (DDR, DDR2, DDR3, DDR4, DDR5) and Flash.
* **SPI / I2C**: Serial communication protocols standard for interfacing external EEPROM/Flash.
* **ONFI (Open NAND Flash Interface)**: Standardized specification for NAND flash memory chips.
* **PCMCIA / SD / eMMC**: Industry standards for portable/embedded memory form factors.

## 2. Comprehension (Concepts & Relationships)

### Visual Interpretation

* **Memory Hierarchy Triangle**:
* **Top (Apex)**: Registers, Cache (Highest speed, lowest capacity, highest cost/bit).
* **Bottom (Base)**: Magnetic/Solid-State Drives (Lowest speed, highest capacity, lowest cost/bit).


* **Read/Write Timing Diagram**:
* **Address Valid**: MPU places address on bus.
* **$\overline{CS}$, $\overline{RD}$, $\overline{WR}$ Asserted**: Active-low control signals drop.
* **Data Valid (Access Time)**: Time delay from valid address/control to stable data on data bus.


* **Address Decoding Block Diagram**: Higher-order MPU address lines $\rightarrow$ Decoder input $\rightarrow$ Decoder output $\rightarrow$ Memory chip $\overline{CS}$ pin.

### Pros vs. Cons: Memory Technologies

| Feature | SRAM | DRAM | Flash (NAND) | Flash (NOR) |
| --- | --- | --- | --- | --- |
| **Storage Element** | 6 Transistors (Flip-Flop) | 1 Transistor + 1 Capacitor | Floating Gate (Pages/Blocks) | Floating Gate (Random Access) |
| **Speed** | Fastest | Moderate | Slow (Write/Erase) | Fast Read, Slow Write |
| **Density/Cost** | Low Density / High Cost | High Density / Low Cost | Very High Density / Low Cost | Moderate Density / High Cost |
| **Application** | CPU Cache (L1/L2) | Main System RAM | Mass Storage (SSD, USB) | Firmware/BIOS execution (XIP) |

### Exam Traps

* **Bit vs. Byte Capacity**: A "$1024 \times 8$" memory chip holds $1024$ locations of $8$ bits each ($1\text{ KB}$ total). A "$1024 \times 1$" chip requires $8$ chips in parallel to form a $1$-byte data bus.
* **Address Lines Miscalculation**: $16$ address lines $= 2^{16} = 65,536$ addresses (often trapped as $64,000$ instead of $64\text{ KB}$).
* **Word Size Confusion**: MPU "word size" equals data bus width. A 32-bit MPU accesses $4\text{ Bytes}$ per memory read.
* **NAND vs. NOR Execution**: Code cannot execute directly from NAND flash; must be copied to RAM. NOR flash supports eXecute In Place (XIP).

### Bottom-Line Summary

The memory subsystem is a hierarchically structured architecture designed to balance the physical trade-offs between speed, capacity, and cost, utilizing ultra-fast SRAM near the processing core to prevent data starvation, volatile DRAM for expansive working space, and non-volatile Flash/ROM for persistent firmware storage, all orchestrated by precise address decoding and strict timing protocols.

## 3. Application (Calculations & Engineering)

### Formulas

* **Total Memory Capacity ($C$)**
* $C = 2^A \times D$
* $A$: Number of address lines
* $D$: Number of data lines (bits per location)
* *Unit*: Bits (divide by 8 for Bytes)
* *Meaning*: Physical storage limit of the memory chip or system.


* **Address Line Calculation ($A$)**
* $A = \log_2 \left( \text{Number of Words} \right)$
* *Unit*: Unitless (Number of pins/wires)
* *Meaning*: Necessary MPU address pins to uniquely select every memory location.


* **Average Memory Access Time (AMAT)**
* $\text{AMAT} = t_{hit} + (p_{miss} \times t_{penalty})$
* $t_{hit}$: Cache hit time
* $p_{miss}$: Miss rate ($1 - \text{Hit Rate}$)
* $t_{penalty}$: Miss penalty (time to fetch from main memory)
* *Unit*: Nanoseconds ($\text{ns}$)
* *Meaning*: Statistical average time to fetch a word, accounting for cache efficiency.


* **Number of Chips Required ($N$)**
* $N = \frac{\text{Total Required System Capacity}}{\text{Capacity of One Chip}}$
* *Unit*: Integer
* *Meaning*: Component count for building a memory module.



### Practical Use

* **Embedded IoT (e.g., ESP32)**: Utilizes external SPI Flash memory combined with internal SRAM; employs memory-mapped eXecute In Place (XIP) to run complex code without exceeding internal volatile limits.
* **Desktop PCs**: Utilizes DIMM (Dual In-line Memory Module) slots populated with DDR4/DDR5 DRAM chips; managed by an integrated memory controller on the CPU die.
* **Automotive ECUs**: Heavy reliance on EEPROM for storing calibration parameters and odometer readings due to byte-level rewritability and extreme temperature tolerance.

### Design Trade-Offs

* **SRAM vs. DRAM Area Penalty**: Replacing DRAM with SRAM increases speed drastically but consumes roughly $6\times$ the silicon footprint, driving up chip manufacturing costs.
* **SLC vs. MLC/TLC Flash**: Single-Level Cell (1 bit/cell) offers highest endurance and speed. Multi/Triple-Level Cell ($2$/$3$ bits/cell) sacrifices speed and lifespan for significantly cheaper, higher-density storage.
* **Harvard vs. Von Neumann Bottleneck**: Separating program memory from data memory (Harvard) eliminates the fetch/execute bus collision, but requires doubling the memory interface pins on the microcontroller package.
